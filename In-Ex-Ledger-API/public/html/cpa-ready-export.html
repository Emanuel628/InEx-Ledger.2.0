Export workflow

CPA-ready bookkeeping exports should explain the record, not just dump the record.

A useful export does more than total the ledger. It should show what is ready, what is missing, what was excluded on purpose, and where the support trail still needs attention.

That is the direction InEx Ledger is built around: review-ready PDF and CSV workpapers with blockers, warnings, support coverage, and final package context.

Why this matters

The export is where bookkeeping quality becomes visible

Missing support stops hiding

The package can show when receipt, mileage, or support coverage is still incomplete.

Excluded items stay separated

Transfers, owner draws, refunds, and similar non-P&L rows can be called out instead of muddying the expense story.

Final review state stays visible

The export can reflect whether the package is still draft, workpaper, or ready for a cleaner handoff.

What a better export includes
  • Transaction totals and category summaries
  • Receipt and support coverage by scope
  • Unresolved exceptions and open review items
  • Excluded items with a reason trail
  • Snapshot and stale-state awareness when the underlying data changes later
